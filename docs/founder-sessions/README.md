# Founder Sessions

Raw founder-session captures live here.

Use this folder for exact founder wording from brainstorming, product clarifications, bug-fix instructions, UI/design discussions, and implementation details that should not be compressed into memory-only summaries.

Founder-session notes are context artifacts, not owner specs. When a captured clarification changes durable behavior, promote the rule into the relevant owner spec under `docs/specs/**` and keep this file as the raw evidence trail.

Each session note should separate:

- `Clarifications` — exact bug-fix behavior, edge cases, terminology corrections, acceptance details, and small implementation details.
- `Feature Sets` — larger feature concepts, architecture ideas, UI systems, workflows, and future build packages.

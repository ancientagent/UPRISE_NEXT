# Surface Contract — Home R1

**Status:** `active`
**Owner:** `founder + codex`
**Last Updated:** `2026-04-05`

## Purpose
Define the minimum stable contract for the `Home` surface so future agents stop collapsing it into `Plot` or treating `Plot` as a peer route.

## Core Rule
- `Home` is the **left** bottom-nav destination.
- `Discover` is the **right** bottom-nav destination.
- `Plot` lives **inside** `Home`.
- `Plot` is the tabbed sectional system within the `Home` surface.

`Home` and `Discover` are peers in the shell.
`Plot` is not a peer to `Home`; it is a structural subsystem of `Home`.

## Primary Job
- Act as the user’s primary anchored surface for current scene participation.
- Hold the `Plot` sectional system.
- Preserve persistent player context.
- Preserve Home Scene identity and current tuned/listening context.

## Secondary Job
- Surface current identity/support context without turning into a vanity profile page.
- Provide stable access to the current Home-side participation system.

## Governing Context
- The persistent player remains a governing system on Home.
- Home Scene identity remains `city + state + music community`.
- Home must inherit current player/listening context rather than creating a parallel context system.

## Structural Contract
Home contains:
1. persistent shell context
2. player context
3. `Plot` as the Home-side tabbed participation system

Home does not require a separate top-level tab set that competes with Plot.

## What Must Remain True
- The left nav destination is `Home`.
- Home is the container for the Home-side participation system.
- Plot remains inside Home.
- Player context remains relevant on Home.
- Home Scene identity remains structurally visible in the Home-side experience.

## What Home Must Not Become
- not a standalone social profile page
- not a generic dashboard
- not a peer competitor to Plot
- not a second Discover surface
- not a creator-vanity homepage

## Known Drift Patterns
- treating `Plot` as the Home route itself
- describing `Plot` and `Home` as separate sibling destinations
- removing player relevance from Home
- turning Home into a generic profile/feed hybrid detached from scene logic

## Current Relationship To Other Surfaces
- `Home` contains `Plot`
- `Discover` is a separate bottom-nav destination
- `Community` is entered explicitly through community links/visit flows, not as a bottom-nav peer

## Open Areas (Not Locked Here)
This contract does not finalize:
- exact Home visual composition outside the Home->Plot relationship
- expanded profile/body behavior beyond existing locked docs
- final Home header/card arrangement
- future V2 Home-only modules

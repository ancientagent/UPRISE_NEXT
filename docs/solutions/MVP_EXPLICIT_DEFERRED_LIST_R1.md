# MVP Explicit Deferred List R1

Status: Active
Owner: Founder + product engineering
Last updated: 2026-04-10

## Purpose
Provide one explicit authority list for what is **not** part of the current MVP.

This document exists so agents do not have to infer deferred scope from scattered roadmap notes, handoffs, or older specs.

If an item appears here, treat it as:
- real product scope if noted
- but not part of the current MVP unless explicitly reactivated in-thread or by a newer founder lock

## Current MVP Core
The active MVP is focused on:
- community creation / resolution / assignment
- RaDIYo and listening-context systems
- minimum revenue surfaces:
  - artist promotion packages
  - event promotion packages

Everything below is outside that core unless explicitly stated otherwise.

## Explicitly Deferred From MVP
### 1. Broader social layer
- broader social-product expansion
- generic social-network behavior
- social as a primary product pillar

### What Counts As Social
For current MVP boundary decisions, treat these as part of the broader social layer:
- message boards
- group or sect discussion channels
- broader community discussion threads
- listener-to-listener social posting
- direct messaging
- vibe / taste match systems
- search parties
- listening rooms
- richer group-coordination / community-conversation tools
- broader people-following / social-activity visibility systems
- generic social-network behaviors

Do not treat these as the broader social layer:
- community creation / resolution / assignment
- RaDIYo and listening-context systems
- artist promotion packages
- event promotion packages
- basic source follow as awareness
- support reactions
- lightweight discovery mechanics already locked for MVP
- sect realization into Uprise/sub-community behavior already explicitly locked

### 2. Causes
- causes are not part of the current MVP surface
- causes are later-version scope, likely V2
- do not build dedicated public cause surfaces in MVP

### 3. National active surface scope
- `national` Discover surface behavior
- `national` Discover search
- active `national` Plot surface behavior beyond broader-platform retained context

### 4. Top 40 / billboard-style lists
- Top 40 UI
- billboard-style ranking lists
- population-dependent leaderboard surfaces

### 5. Metrics-as-authority systems
- activity points as authority
- activity score as authority
- predictive or governance-weighted scoring systems
- recommendation-engine behavior derived from descriptive stats

### 6. Later-version domain surfaces
These remain legitimate product domains, but are not current-MVP-driving:
- ambassadors
- venues
- mixologists
- mixes

### 7. Broader cause/civic extension surfaces
- cause visibility surfaces
- cause runtime expansion
- non-essential civic-extension systems beyond current sect/Uprise realization needs

### 8. Full social/community extension surfaces
- search parties
- listening rooms
- broader message-board/group expansion beyond current MVP need

### 9. Full Print Shop / artifact expansion
- full Print Shop runtime as a complete surface/system
- internal artifact issuance mechanics beyond what current MVP explicitly requires

### 10. Promo section / business promotions wall
- the promo section / promotions wall is not part of the current MVP core
- treat it as a later simple business surface
- current founder direction:
  - it is effectively a wall of JPEG-style business promos
  - businesses may post there from the web layer
  - businesses may not need listener accounts for that later surface

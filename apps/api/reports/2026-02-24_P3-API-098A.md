# P3-API-098A — Registrar project submission primitive

## Summary
Implemented `POST /registrar/project` as an additive registrar submission primitive with baseline `submitted` lifecycle status.

## Implementation Notes
- Added `ProjectRegistrationSchema` with payload:
  - `sceneId` (uuid)
  - `projectName` (trimmed, non-empty, max 140)
- Added controller route:
  - `POST /registrar/project`
- Added service method:
  - `submitProjectRegistration(userId, dto)`
  - Enforces existing registrar Home Scene guardrails:
    - target scene must exist
    - scene must be city-tier
    - user must exist
    - scene must match requester Home Scene tuple
  - Persists `RegistrarEntry` with:
    - `type = project_registration`
    - `status = submitted`
    - payload `{ projectName }`

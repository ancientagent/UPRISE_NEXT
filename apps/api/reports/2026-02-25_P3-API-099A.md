## Implementation Notes
- Added `SectMotionRegistrationSchema` payload:
  - `sceneId` (uuid)
- Added controller route:
  - `POST /registrar/sect-motion`
- Added service method:
  - `submitSectMotionRegistration(userId, dto)`
  - Enforces existing registrar guardrails:
    - target scene must exist,
    - scene must be city-tier,
    - requester user must exist,
    - target scene must match requester Home Scene tuple.
  - Persists `RegistrarEntry` with:
    - `type = sect_motion`
    - `status = submitted`
    - payload `{}` (state-machine/schema details intentionally deferred).
